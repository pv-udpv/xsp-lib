## Design Principles

### Separation of Concerns

Each layer has a single responsibility:
- **Transport**: Network I/O and connection management
- **Upstream**: Protocol-specific logic and error mapping
- **Parser**: Data transformation and version filtering

### No Global State

All dependencies are explicitly passed:
- No singleton HTTP clients
- No context variables
- Everything is testable and mockable

### Version Awareness

Both macros and parser respect VAST versions:
- Automatic filtering prevents forward compatibility issues
- Backward compatibility enables gradual upgrades
- Version metadata preserved for debugging

### Error Transparency

Exceptions map clearly through layers:
```
httpx â†’ TransportError â†’ VastError â†’ Application
```

Each layer adds context without hiding the root cause.
